# MindScript Docs

Welcome to the canonical entry point for MindScript documentation. (Formerly OpenSpec.)

## Start here

1. **Get oriented:** Read the [MindScript overview](mindscript/spec-overview.md) to understand the spec-first philosophy and core concepts.
2. **See it in action:** Walk through the [Quickstart](mindscript/quickstart.md) to build your first Context + Turn.
3. **Integrate quickly (default):** Start with [MCP-native](mindscript/consumer-quickstart-mcp.md).
4. **Fallback path:** Use [schema-first / no-MCP mode](mindscript/consumer-quickstart-schema.md) when MCP plumbing is not available yet.
5. **Go deeper:** Learn the [spec language](mindscript/spec-language.md), [validation model](mindscript/spec-validation.md), and [adoption charter](mindscript/adoption-charter.md).
6. **Integrate at scale:** Explore [integrations](integrations/bdd/@software/README.md) and adapters for your tooling.

## Documentation map

- **MindScript Spec** — The canonical spec language, schema, templates, and verification model. Start with the [spec overview](mindscript/spec-overview.md).
- **Adoption** — Public package policy, migration map, and extension rules. Start with the [adoption charter](mindscript/adoption-charter.md).
- **Integrations** — Optional adapters, templates, and reference examples for real-world stacks like `@software` and BDD. Start with the [BDD integrations overview](integrations/bdd/@software/README.md).
- **API Reference** — Generated TypeDoc references for the TypeScript packages. Jump to the [runtime API docs](api/runtime/README.md).
- **Legacy** — Historical OpenSPI/OpenSpec pages preserved for compatibility. See [legacy overview](openspi/overview.md).
