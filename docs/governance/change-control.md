# Change Control

## RFC Required

An RFC is required for:

1. breaking core schema changes
2. new normative profiles
3. MCP tool-name or error-code changes
4. compatibility policy changes
5. deprecation window changes

## Additive-First Rule

Core changes should be additive by default.

If a change is not additive, the proposal must include:

- motivation
- migration plan
- compatibility impact
- rollout guidance
